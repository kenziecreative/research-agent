# Pattern Recognition Guide

Types of patterns to look for when cross-referencing research sources. Not every cross-reference run will find all pattern types — report only what the evidence supports.

## 1. Convergence

Multiple independent sources reach the same conclusion through different evidence or methodology. This is the strongest signal in cross-referencing.

**How to confirm independence:** Trace each source to its original data. If two articles cite the same underlying study, that is one data point reported twice, not convergence. True convergence requires different underlying evidence — e.g., an industry report using survey data and a case study using interviews both pointing to the same conclusion.

**What to report:** The converged finding, which sources support it, and whether the sources are genuinely independent.

## 2. Contradiction

Sources directly disagree on a specific claim — not ambiguity, but incompatible assertions about the same fact or figure.

**How to present:** Name both sources and their specific claims. Do not resolve the contradiction — present both sides and note any differences in methodology, time period, or scope that might explain the disagreement.

**When to investigate further:** When the contradiction involves a core question for the current phase. Peripheral contradictions can be noted without deep investigation.

## 3. Gap Clusters

Multiple sources reference a topic or assume a fact but none provide actual evidence for it. The gap is significant precisely because multiple sources expected evidence to exist.

**How to distinguish from "not relevant":** A gap cluster is when sources talk around something without providing data. If no sources mention a topic at all, it may simply be outside the research scope.

**What to report:** The topic that sources reference but do not evidence, which sources reference it, and whether the gap is researchable (public sources likely exist but were not collected) or structural (the data may not exist publicly).

## 4. Temporal Trends

A claim that was true in older sources but is contradicted or updated by newer ones, suggesting the landscape has shifted.

**How to date-stamp:** Note the publication date or data year of each source contributing to the pattern. Present the shift as a trajectory, not as "old source was wrong."

**What to report:** The original claim with dates, the updated claim with dates, and the direction of change. This is especially valuable for market data, technology adoption, and regulatory landscapes.

## 5. Source-Type Skew

All evidence for a claim comes from one type of source — e.g., only vendor marketing, only academic research, only press coverage. Missing source diversity weakens confidence even when multiple sources agree.

**What to flag:** The claim, the source types that support it, and what source types are missing. Example: "All three sources for the 40% adoption rate are vendor-published. No independent survey or analyst report corroborates this."

**When it matters most:** For quantitative claims, market size estimates, adoption rates, and effectiveness claims. These are the areas where source type most strongly predicts bias.

## 6. Outlier Signals

A single source making a claim that no other source supports or contradicts. Not wrong by default, but worth flagging for awareness.

**When to flag:** When the outlier claim is relevant to a core question and the source has reasonable credibility. Outliers from low-credibility sources can be noted but need not be featured.

**When to ignore:** When the outlier is peripheral to the research questions or comes from a source with clear bias on the topic.

## Pattern Strength Assessment

| Evidence Level | Label | Meaning |
|---|---|---|
| 1 source | Claim | A single assertion, not yet a pattern |
| 2 independent sources | Emerging | Worth noting, not yet robust |
| 3+ independent sources | Pattern | Reliable enough to build on in synthesis |
| 3+ dependent sources (same underlying data) | Echo | Appears strong but is actually one data point repeated |
