# research-agent

The source of truth for the `/knz-research` Claude Code command — a structured AI-assisted research framework with state management, evidence standards, and integrity enforcement.

## What it does

`/knz-research` scaffolds a complete research project for one of 8 research types:

- PRD Validation
- Exploratory Thesis
- Competitive Analysis
- Company Research (For-Profit)
- Company Research (Non-Profit)
- Person Research
- Market/Industry Research
- Presentation Research

Each project gets a full directory structure, `CLAUDE.md`, `STATE.md`, reference protocols, source registry, gap tracker, cross-reference file, and a type-specific research plan generated by an agent.

## Files

| File | Purpose |
|------|---------|
| `knz-research.md` | The command itself — all scaffolding logic, type templates, agent definitions, and project-level command specs |

## Installation

The command is installed via symlink so edits here take effect immediately:

```
~/.claude/commands/knz-research.md -> [this repo]/knz-research.md
```

To set up on a new machine:

```bash
ln -s /path/to/research-agent/knz-research.md ~/.claude/commands/knz-research.md
```

## How it works

When invoked, `/knz-research` asks three questions (research type, topic, target directory), then:

1. Creates the project directory structure
2. Launches an agent to generate a type-specific research plan with phases and questions grounded in preliminary web research
3. Assembles `CLAUDE.md` with the relevant type's finding tags, credibility hierarchy, and enforcement rules
4. Writes reference files (`source-standards.md`, `writing-standards.md`, `tools-guide.md`, `canonical-figures.json`)
5. Creates project-level commands: `/process-source`, `/cross-ref`, `/check-gaps`, `/summarize-section`, `/audit-claims`
6. Creates the `research-integrity` agent for post-write drift detection

## Key design principles

- **canonical-figures.json** — Single source of truth for all cross-phase numbers. Prevents data drift as findings move from phase outputs into synthesis.
- **research-integrity agent** — Runs after every write to catch fabricated data, range narrowing, qualifier stripping, and cross-phase drift before they propagate.
- **Project boundary rule** — All file writes stay within the research project directory. No writes to other projects or system paths, even if a user request points elsewhere.
- **Mandatory gates** — `/cross-ref` every 5-8 sources. `/check-gaps` before each new phase. `/audit-claims` before any draft reaches `outputs/`. Phase completion requires all four checks in STATE.md.
