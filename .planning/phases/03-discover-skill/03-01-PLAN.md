Create `.claude/commands/research/discover/SKILL.md` following the established skill pattern from process-source/SKILL.md. The skill must contain these sections in order:

**Frontmatter (YAML):**
- name: discover
- description: "Execute type-aware, multi-channel source discovery for the current research phase"
- argument-hint: (omit — no arguments required; optionally support `--channel {name}` per Claude's discretion from CONTEXT.md)
- allowed-tools: [tavily_search, Bash, WebSearch, tavily_extract, Read, Grep, Glob]
- disable-model-invocation: true

**# /research:discover**

**## Pre-checks (mandatory before execution):**
1. Read `research/STATE.md` — extract current active phase name. If no active phase, error: "No active phase. Run `/research:start-phase` first."
2. Read `research/research-plan.md` — extract the research questions for the active phase. These become query inputs.
3. Determine the project's research type from CLAUDE.md (the research-type field).
4. Read the type-channel map at `.claude/reference/discovery/type-channel-maps/{research-type}.md` — parse `active-channels` from frontmatter and find the Discovery Group whose Phases list matches (or partially matches) the current phase name.
5. If no Discovery Group matches the current phase, report: "No discovery channels mapped for this phase — this phase uses existing sources." and stop.
6. Check if `research/discovery/{phase}-candidates.md` already exists. If it does, new results will be appended with a timestamp separator (per CONTEXT.md decision).

**## Process:**

Step-by-step orchestration flow:

1. **Resolve channels:** From the matched Discovery Group, collect Primary channels and Secondary channels. Execute primary channels first.

2. **For each channel in priority order:**
   a. Print status line: "{Channel Name}: querying..."
   b. Read the channel playbook at `.claude/reference/discovery/channel-playbooks/{channel-name}.md`
   c. Select the appropriate query template from the playbook based on the research phase context (topic search for broad phases, entity search for company/person phases, etc.)
   d. Substitute placeholders in the template: `{topic}` = research subject from research-plan.md, `{company_name}` or `{entity_name}` = research subject name, `{year_minus_2}` / `{year_minus_5}` = computed from current date, `{email}` = "research-agent@example.com" for OpenAlex polite pool
   e. Execute the query using the tool specified in the playbook
   f. Cap results at 8 sources maximum per channel (per DSKL-07)
   g. Print status line: "{Channel Name}: found {N}" or "{Channel Name}: error — {reason}" or "{Channel Name}: skipped (not in active channels)"
   h. If the channel fails (timeout, rate limit, API error), log the failure, continue to next channel

3. **Degradation handling (per CONTEXT.md decisions):**
   - Tavily unavailable: switch to WebSearch-only mode automatically, label all results "via WebSearch (Tavily fallback)", print warning
   - The attempt IS the check — no pre-flight availability detection
   - Absolute floor: WebSearch with domain-scoped queries from playbooks. If even WebSearch fails, produce empty candidates file with all-channels-failed status report

4. **Deduplicate:** Before writing, check if `research/discovery/{phase}-candidates.md` already exists. If so, deduplicate new results against existing entries by URL. Note "N duplicates skipped" in summary.

5. **Determine source status for each candidate:**
   - For Tavily results with full content snippets: mark ACCESSIBLE
   - For URL-only results (no content verified): mark DISCOVERED
   - For Google Patents constructed URLs: mark DISCOVERED (extraction needed)
   - For results from sites known to be open access: mark ACCESSIBLE
   - For paywalled domains (wsj.com, ft.com, bloomberg.com): mark DISCOVERED

6. **Write candidates file** at `research/discovery/{phase}-candidates.md`:

   Format:
   ```
   # Discovery Candidates: {Phase Name}

   **Research type:** {type}
   **Phase:** {phase name}
   **Discovered:** {timestamp}

   ## Summary

   | Channel | Status | Sources |
   |---------|--------|---------|
   | Web Search | found 6 | 6 |
   | Financial | found 3 | 3 |
   | Social Signals | skipped (not in active channels) | 0 |
   | Academic | found 5 | 5 |
   | Regulatory | error (timeout) | 0 |
   | Domain-Specific | found 2 | 2 |

   **Total candidates:** {N}
   **Duplicates skipped:** {N}

   ## Web Search

   - [ACCESSIBLE] Title of source — https://example.com/article
     Brief snippet describing what this source contains

   - [DISCOVERED] Title of source — https://example.com/paywalled
     Brief snippet describing what this source contains

   ## Financial
   ...

   ## Academic
   ...
   ```

   If appending to existing file, add a separator:
   ```
   ---
   ## Re-discovery: {timestamp}
   ```
   Then append new sections below.

7. **Print completion summary and next steps:**
   "Discovery complete. {N} candidates found across {M} channels. Review candidates in research/discovery/{phase}-candidates.md. Process selected sources with `/research:process-source <url>`."

**## Channel Execution: Tavily-Based Channels**

Document specific execution instructions for each Tavily-based channel:

**Web Search (channel-type: web-search):**
- Use tavily_search with parameters from web-search.md query templates
- topic: "general" for broad phases, "news" for recent-developments phases (add days=90)
- include_domains: leave empty for general web search
- Fallback: WebSearch with same query text

**Financial (channel-type: financial):**
- Use tavily_search with parameters from financial.md query templates
- topic: "finance" for market/earnings queries
- include_domains: per template (sec.gov, finance.yahoo.com, bloomberg.com, reuters.com, etc.)
- Note: wildcard domains (investor.*.com) are unconfirmed in Tavily — use explicit domains if wildcards fail
- Fallback: WebSearch with site: operators

**Social Signals (channel-type: social-signals):**
- Use tavily_search with parameters from social-signals.md query templates
- include_domains: reddit.com, news.ycombinator.com, stackoverflow.com, twitter.com, linkedin.com, medium.com
- Note: wildcard community.*.com is unconfirmed — use explicit community domains if available
- Fallback: WebSearch with site: operators

**Domain-Specific (channel-type: domain-specific):**
- Read the type-channel map's "Domain-Specific Sources" section to identify which type hooks apply
- Execute only the hooks specified for the current research type
- Patent Search hook: Construct Google Patents URL, attempt tavily_extract. If tavily_extract unavailable, log the URL as DISCOVERED with note "extraction unavailable — manual review required"
- Industry Databases hook: Use tavily_search with domain scoping per hook template
- Educational Resources hook: Use tavily_search per hook template (curriculum-research type only)
- Specialized Registries hook: Use tavily_search per hook template (person-research type only)
- Fallback: WebSearch with site: operators for database domains

**## Channel Execution: HTTP API Channels**

Note: "Academic and regulatory channel execution details are in the full playbooks at `.claude/reference/discovery/channel-playbooks/academic.md` and `regulatory.md`. Read those playbooks at execution time for curl templates, response parsing, and degradation instructions."

Briefly document:

**Academic (channel-type: academic):**
- Primary: Bash curl to OpenAlex API (see academic.md for templates)
- Extract: title, DOI, authors, citation count, open-access status, OA URL
- Status: is_oa=true → ACCESSIBLE, is_oa=false → DISCOVERED
- Fallback: tavily_search with academic domain scoping, then WebSearch

**Regulatory (channel-type: regulatory):**
- Primary: Bash curl to EDGAR EFTS and ProPublica APIs (see regulatory.md for templates)
- EDGAR: Extract filing date, form type, entity name, construct filing URL from accession number
- ProPublica: Extract EIN, name, revenue, construct 990 profile URL
- All SEC filings are ACCESSIBLE by law
- Fallback: tavily_search with sec.gov / propublica.org domain scoping, then WebSearch

**## Guardrails:**
1. Discovery output goes to `research/discovery/` ONLY. Never write to `research/notes/`, `research/sources/registry.md`, or any other research file. That boundary belongs to process-source.
2. Never auto-feed discovered sources into process-source. The candidate list is for human review.
3. Cap results at 8 sources per channel per query. If an API returns more, take the top 8 by relevance/citation count.
4. Always include the User-Agent header for SEC EDGAR requests per SEC policy.
5. Include mailto parameter for OpenAlex polite pool (10 req/s tier).
6. Do not run all domain-specific hooks — only the hooks specified by the research type's channel map.
7. When a channel fails, continue with remaining channels. Never abort discovery due to a single channel failure.
8. Label all degraded/fallback results inline with the actual method used.

**## Common Failure Modes:**

| Failure Mode | Prevention |
|---|---|
| Running discovery with no active phase | Pre-check reads STATE.md. If no active phase, error with guidance before any channel execution. |
| Querying channels not relevant to the research type | Read the type-channel map first. Only execute channels listed in the matched Discovery Group. |
| Overwriting prior discovery work on re-run | Check for existing candidates file. If exists, append with timestamp separator and deduplicate by URL. |
| Feeding discovered sources directly into processing pipeline | Guardrail: output to research/discovery/ only. Suggest process-source in completion message but never invoke it. |
| Silent channel failure with no status reporting | Print per-channel status lines during execution. Include all channels in summary table with explicit status. |
| Exceeding API rate limits | Follow rate limits from playbooks: EDGAR max 5 req/s, OpenAlex include mailto for 10 req/s, Tavily ~1000/month. |
| Missing User-Agent for EDGAR | Guardrail: always include User-Agent header. Embedded in curl template. |

Print the completion summary with candidate count, channel statuses, and next-step guidance. The candidates file is the primary artifact.
