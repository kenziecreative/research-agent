<!-- Key structures the executor needs from claim-graph.json schema (defined in step 8b): -->

Each claim node in claim-graph.json has:
- id: string (e.g. "c001-market-size-exceeds-four")
- text: string (claim text)
- phase: number (current phase)
- section: string (section name)
- confidence_tier: "High" | "Moderate" | "Low" | "Insufficient"
- source_files: string[] (note filenames)
- figure_ids: string[] (IDs from canonical-figures.json, may be empty)
- evidence_directness: "Direct" | "Indirect"
- source_count: number

canonical-figures.json entries have:
- id: string
- value: number | string
- unit: string
- qualifier: string
- source_phase: number
- source_file: string
- confidence: string
- registered_when: string
- referenced_by: string[]

New field to add (drift_warning):
{
  "figure_id": "<id from figure_ids>",
  "expected_value": <value stored in claim at last audit>,
  "canonical_value": <current registry value>
}
