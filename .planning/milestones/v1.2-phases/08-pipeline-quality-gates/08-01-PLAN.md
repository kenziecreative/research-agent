    - **customer-safari.md**: 2 years — user behavior and preferences shift with market conditions
    - **curriculum-research.md**: 5 years — educational content and pedagogy evolve slowly; foundational material is durable
    - **opportunity-discovery.md**: 2 years — market opportunities are time-sensitive
    - **person-research.md**: 3 years — career arcs move slowly but roles and affiliations change
    - **presentation-research.md**: 2 years — presentation topics typically address current issues

  <action>
    Read the current summarize-section SKILL.md. Add a new pre-check 4 (after the existing contradiction gate pre-check 3) that warns about stale sources before synthesis begins.

    Pre-check 4 logic:
    1. Read the research type template (from research/research-plan.md which names the type) to get the Staleness Threshold value
    2. Read all source notes in research/notes/ relevant to this section
    3. For each source note, compare the data year (not publication year — a 2025 article citing 2021 data uses 2021) against the current year minus the staleness threshold
    4. If any sources exceed the threshold, display a staleness warning listing each stale source with its data year and how many years over threshold it is

    Warning format:
    ```
    Source staleness advisory (threshold: N years for [research type]):

    - [source-note-filename]: data year [YYYY], [M] years over threshold
    - [source-note-filename]: data year [YYYY], [M] years over threshold

    These sources will be included in synthesis with age caveats. Stale evidence is still evidence but carries reduced weight.
    ```

    This is a WARNING, not a gate — synthesis proceeds after displaying the advisory. Do not block.

    Also add a corresponding entry to the Common Failure Modes table:
    | Failure Mode | Prevention |
    | Treating stale sources as equally current — using old data without age caveat | Check each source's data year against the type's staleness threshold. If stale, include the finding but add an age caveat: "Based on [YYYY] data..." so the reader knows the evidence may not reflect current conditions. |

    Also add guardrail 8: "When a source exceeds the staleness threshold, include its findings in the draft but add an explicit age caveat noting the data year. Do not silently present stale data as current."
  </action>
