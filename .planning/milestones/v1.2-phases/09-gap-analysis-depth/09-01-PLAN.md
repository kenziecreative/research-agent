  <action>
Update the coverage-assessment-guide.md to add the three-tier match classification system. This guide is the single source of truth for coverage assessment — check-gaps references it.

Changes to make:

1. Add a new section "## Match Classification" after "## What 'Addressed' Actually Means" that defines:
   - **Direct:** Source contains a specific claim, data point, or argument that substantively answers the question. The source's evidence directly addresses the question's subject matter.
   - **Adjacent:** Source addresses a related topic but does not answer the specific question. Example: a source about "cloud infrastructure market size" is adjacent to a question about "AWS market share" — related but not the same question. Display with explanation: "Addresses [actual topic] rather than [phase question]"
   - **None:** Source does not address the question or any related topic.

2. Update the "What 'Addressed' Actually Means" section:
   - Under "A question is NOT addressed when" — change the bullet "A source addresses a related but different question (adjacent is not the same as addressed)" to reference the new Adjacent classification: "A source addresses a related but different question — classify as Adjacent, not Direct (see Match Classification)"

3. Update Coverage Status Definitions to incorporate independence:
   - **Complete:** 3+ **independent** sources with **Direct** matches, no unresolved contradictions, source types are mixed
   - **Partial:** 1-2 **independent** sources with Direct matches, or sources exist but coverage is thin, one-sided, or source-type skewed
   - **Not started:** No source notes have a **Direct** match for this question (including questions with only Adjacent matches)
   - **Addressed but unbalanced:** **Direct** sources exist but represent only one perspective or source type
   - Add note: "Adjacent matches do not contribute to coverage status. A question with 3 Adjacent sources and 0 Direct sources is Not Started."

4. Add a new section "## Source Independence" before "## When to Accept Gaps" that documents:
   - Independence is determined by the origin_chain field in source notes (added in Phase 7)
   - Sources sharing the same origin (e.g., multiple articles all citing the same original study) collapse to one independent data point
   - Coverage status uses independent source count: 3 sources sharing one origin = 1 independent = Partial, not Complete
   - Cross-reference with pattern-recognition-guide.md Echo level for shared-origin clusters

5. Update Coverage Assessment Workflow (step list at bottom):
   - Step 1: List source notes that address the question — classify each as Direct, Adjacent, or None
   - Step 2 (updated): For Direct sources, verify evidence is substantive
   - Step 3 (updated): Check source independence using origin_chain — collapse shared-origin sources
   - Steps 4-6: unchanged
   - Step 7 (new): Assign coverage status using independent Direct source count only

2. **Read all files in `research/notes/`** to understand what's been covered. For each source note, extract:
   - The origin_chain field (primary vs. secondary, cited originals)
   - The specific claims, data points, and arguments the source provides

6. **For each phase in the research plan, for each question:**

   c. For Adjacent matches: note with a one-line explanation: "Addresses [actual topic] rather than [phase question]"
   d. Assign coverage status using the coverage assessment guide definitions — based on independent Direct source count only. Adjacent matches do not contribute to coverage status.
   e. Flag lopsided coverage: any question with only 1 independent Direct source gets a lopsided flag.

   - **Direct coverage:** N questions (N%)

   #### Q: [Question text]

   - [source-note-filename] [Source: citation] — [brief evidence summary]

   - [source-note-filename]: Addresses [actual topic] rather than [phase question]

8. Lopsided coverage flag triggers at exactly 1 independent Direct source — not 0 (that is Not Started) and not 2+ (that is adequate for non-Complete status).

| Missing lopsided coverage on central questions | After assigning coverage status, scan for any question with exactly 1 independent Direct source. Single-source coverage on a phase's central question is a gap worth flagging explicitly. |
