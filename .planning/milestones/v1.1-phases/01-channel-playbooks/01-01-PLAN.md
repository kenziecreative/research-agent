Create the web-search playbook as a skill file with YAML frontmatter. This is the foundational Tavily channel — the most general-purpose discovery channel and the fallback for other channels.

**YAML frontmatter** (required fields per CONTEXT.md decisions):
- name: web-search
- description: General web discovery via Tavily search
- allowed-tools: [tavily_search, WebSearch]
- channel-type: web-search

**Section layout** (all 6 playbooks use identical headers in this order — this plan establishes the canonical layout):

1. **Channel Overview** — What this channel discovers, when to use it, what it cannot find
2. **Tool Configuration** — Primary tool (tavily_search), parameters, authentication requirements
3. **Query Templates** — 2-3 query variants with exact placeholders. For web-search:
   - Topic search: `tavily_search` with `query: "{topic} {context_keywords}"`, `search_depth: "advanced"`, `max_results: 8`, `topic: "general"`
   - Entity search: `tavily_search` with `query: "{entity_name} {entity_type}"`, `search_depth: "advanced"`, `max_results: 5`, `include_domains: []` (no domain filtering for general entity lookup)
   - News/recent search: `tavily_search` with `query: "{topic} recent developments"`, `search_depth: "advanced"`, `max_results: 5`, `topic: "news"`, `days: 90`
4. **Credibility Tiers** — Channel-level source ranking (label format, no numeric scores):
   - Tier 1 (Highest): Government databases (.gov), peer-reviewed publications, official filings
   - Tier 2 (Reliable): Established news organizations, industry analyst reports, recognized institutions
   - Tier 3 (Contextual): Company websites, press releases, blog posts, user-generated content
   - Red flags: SEO-optimized listicles, content farms, AI-generated aggregation sites, press releases disguised as journalism
5. **Source Status Taxonomy** — Define the three statuses used across all channels:
   - DISCOVERED: URL found, not yet verified accessible
   - ACCESSIBLE: Content confirmed reachable and extractable
   - PROCESSED: Content extracted and integrated into research notes
6. **Degradation Behavior** — Fallback chain:
   - Primary: tavily_search
   - Fallback: WebSearch (if Tavily unavailable) — label results as "via WebSearch (Tavily fallback)"
   - Unavailable criteria: HTTP 5xx, timeout > 30s, rate limit response, authentication failure
   - Without Tavily: WebSearch is the direct replacement; results will be less targeted (no include_domains, no topic filtering)
7. **Rate Limits** — Tavily: per-plan limits (free tier ~1000 searches/month); no per-second throttle needed for typical research sessions

Note: The Source Status Taxonomy in section 5 is the canonical definition. Other playbooks reference this section rather than redefining it (they include a brief note pointing to web-search.md for the full taxonomy).

Create both playbooks using the identical section layout established in Task 1. Both are Tavily-based channels with different domain scoping.

**social-signals.md:**

YAML frontmatter: name: social-signals, description: Social platforms and community discussion discovery, allowed-tools: [tavily_search, WebSearch], channel-type: social-signals

Query Templates (2-3 variants):
- Community discussion: `tavily_search` with `query: "{topic} discussion"`, `search_depth: "advanced"`, `max_results: 8`, `include_domains: ["reddit.com", "news.ycombinator.com", "stackoverflow.com", "community.*.com"]`
- Expert opinion: `tavily_search` with `query: "{topic} {entity_name} review OR opinion OR analysis"`, `search_depth: "advanced"`, `max_results: 5`, `include_domains: ["twitter.com", "x.com", "linkedin.com", "medium.com", "substack.com"]`
- Forum/niche: `tavily_search` with `query: "{topic} forum"`, `search_depth: "basic"`, `max_results: 5`, `topic: "general"`

Credibility Tiers:
- Tier 1: Verified expert accounts, established community contributors (high karma/reputation), official company social accounts for factual announcements
- Tier 2: Industry practitioners with identifiable credentials, moderated forum threads with multiple corroborating responses
- Tier 3: Anonymous users, single-source claims, unverified accounts
- Red flags: Astroturfing patterns (new accounts praising products), engagement-farmed posts, screenshots of claims without links

Source Status Taxonomy: Reference web-search.md for canonical definitions (DISCOVERED / ACCESSIBLE / PROCESSED).

Degradation: Primary tavily_search -> Fallback WebSearch with site-specific queries (e.g., "site:reddit.com {topic}"). Label as "via WebSearch (Tavily fallback)".

Rate Limits: Same Tavily limits as web-search channel.

---

**financial.md:**

YAML frontmatter: name: financial, description: Financial filings, market data, and investment research discovery, allowed-tools: [tavily_search, WebSearch], channel-type: financial

Query Templates (2-3 variants):
- SEC filings search: `tavily_search` with `query: "{company_name} SEC filing 10-K OR 10-Q OR 8-K"`, `search_depth: "advanced"`, `max_results: 5`, `include_domains: ["sec.gov", "investor.*.com", "ir.*.com"]`
- Market/financial data: `tavily_search` with `query: "{company_name} revenue earnings financial results"`, `search_depth: "advanced"`, `max_results: 5`, `include_domains: ["finance.yahoo.com", "bloomberg.com", "reuters.com", "wsj.com", "ft.com", "seekingalpha.com"]`
- Investment analysis: `tavily_search` with `query: "{company_name} analysis valuation"`, `search_depth: "advanced"`, `max_results: 5`, `topic: "finance"`, `include_domains: ["morningstar.com", "fool.com", "seekingalpha.com"]`

Note: This playbook covers Tavily-scoped financial discovery. The regulatory channel (Plan 02) covers direct EDGAR/ProPublica HTTP API access for deeper filing retrieval.

Credibility Tiers:
- Tier 1: SEC filings (10-K, 10-Q, proxy), earnings transcripts, audited financial statements
- Tier 2: Bloomberg, Reuters, WSJ reporting; Morningstar analysis; recognized equity research
- Tier 3: Seeking Alpha contributor posts, financial blogs, company press releases about earnings
- Red flags: Penny stock promotion sites, "guaranteed returns" language, undisclosed paid promotion, outdated financial data presented as current

Source Status Taxonomy: Reference web-search.md for canonical definitions.

Degradation: Primary tavily_search -> Fallback WebSearch with domain-scoped queries. Financial data discovery is heavily web-based, so degradation impact is lower than API-based channels.

Rate Limits: Same Tavily limits as web-search channel.
