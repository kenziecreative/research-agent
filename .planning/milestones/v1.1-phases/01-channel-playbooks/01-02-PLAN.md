Create the academic channel playbook. This channel uses OpenAlex HTTP API as its primary tool (free, no auth required, comprehensive academic coverage). The discover skill will execute these as Bash curl commands.

**YAML frontmatter:**
- name: academic
- description: Academic paper and scholarly source discovery via OpenAlex API
- allowed-tools: [Bash, tavily_search, WebSearch]
- channel-type: academic

**Use the identical section layout** (same 7 headers in same order as the Tavily-based playbooks in Plan 01):

1. **Channel Overview** — Discovers peer-reviewed papers, preprints, conference proceedings, books. Best for research types needing empirical evidence, literature reviews, or technical depth. Cannot find: unpublished work, most dissertations, non-English publications with poor indexing.

2. **Tool Configuration** — Primary: Bash (curl to OpenAlex API). No authentication required. Base URL: `https://api.openalex.org`. Polite pool: include `mailto` parameter with user's email for faster rate limits. Response format: JSON.

3. **Query Templates** (3 variants with exact curl commands):
   - Topic search: `curl -s "https://api.openalex.org/works?search={topic}&filter=cited_by_count:>10&per_page=8&sort=relevance_score:desc&mailto={email}"`
   - Author search: `curl -s "https://api.openalex.org/works?filter=authorships.author.display_name.search:{author_name}&per_page=8&sort=publication_date:desc&mailto={email}"`
   - Filtered search (recent, high-impact): `curl -s "https://api.openalex.org/works?search={topic}&filter=from_publication_date:{year_minus_5},is_oa:true,cited_by_count:>5&per_page=8&sort=cited_by_count:desc&mailto={email}"`

4. **Credibility Tiers:**
   - Tier 1: Papers in peer-reviewed journals with high citation counts (>50), systematic reviews, meta-analyses
   - Tier 2: Recent peer-reviewed papers (<50 citations — may be new), conference proceedings from top-tier venues, preprints from established researchers
   - Tier 3: Preprints without peer review, low-citation papers, conference workshops, book chapters
   - Red flags: Predatory journal indicators (check against known predatory publishers), papers with retraction notices, self-citation inflation, papers from paper mills (multiple papers with suspiciously similar structures)

5. **Source Status Taxonomy** — Reference web-search.md for canonical definitions (DISCOVERED / ACCESSIBLE / PROCESSED). Additional note: OpenAlex `is_oa: true` indicates open access — these are likely ACCESSIBLE. Paywalled papers are DISCOVERED only.

6. **Degradation Behavior:**
   - Primary: Bash curl to OpenAlex API
   - Fallback: `tavily_search` with `query: "{topic} site:scholar.google.com OR site:arxiv.org OR site:pubmed.ncbi.nlm.nih.gov"`, `search_depth: "advanced"`, `max_results: 8` — label as "via Tavily (OpenAlex fallback)"
   - Without Tavily: WebSearch with academic domain keywords — label as "via WebSearch (OpenAlex+Tavily fallback)"
   - Unavailable criteria: HTTP 5xx, timeout > 15s, HTTP 429 (rate limit)

7. **Rate Limits** — OpenAlex polite pool (with mailto): 10 requests/second. Without mailto: 1 request/second. No daily cap for reasonable usage.

**Example Response section** (per CONTEXT.md — include for non-Tavily channels):
Show a trimmed example of the OpenAlex JSON response for a works query, highlighting the fields the discover skill should extract: `title`, `doi`, `publication_date`, `cited_by_count`, `authorships[].author.display_name`, `open_access.is_oa`, `open_access.oa_url`, `primary_location.source.display_name`. Keep the example concise (one result object).

Create the regulatory channel playbook. This channel uses two HTTP APIs: SEC EDGAR EFTS (for-profit filings) and ProPublica Nonprofit Explorer (nonprofit 990s). Both are free, no auth required.

**YAML frontmatter:**
- name: regulatory
- description: Government filings and regulatory document discovery via EDGAR and ProPublica APIs
- allowed-tools: [Bash, tavily_search, WebSearch]
- channel-type: regulatory

**Use the identical section layout** (same 7 headers):

1. **Channel Overview** — Discovers SEC filings (10-K, 10-Q, 8-K, S-1, DEF 14A), nonprofit 990 tax filings, and regulatory documents. Best for company research (for-profit and non-profit), financial analysis, compliance research. Cannot find: private company filings, international regulatory documents, state-level filings.

2. **Tool Configuration** — Primary: Bash (curl). Two APIs:
   - SEC EDGAR EFTS: Base URL `https://efts.sec.gov/LATEST/search-index?q={query}&dateRange=custom&startdt={start}&enddt={end}&forms={form_types}`. Required: User-Agent header with contact info per SEC policy.
   - ProPublica Nonprofit Explorer: Base URL `https://projects.propublica.org/nonprofits/api/v2/search.json?q={org_name}`. No auth required.
   - EDGAR full-text search: `https://efts.sec.gov/LATEST/search-index?q={query}&forms={forms}`

3. **Query Templates** (3 variants):
   - EDGAR company filing search: `curl -s -H "User-Agent: ResearchAgent (contact@example.com)" "https://efts.sec.gov/LATEST/search-index?q=%22{company_name}%22&forms=10-K,10-Q,8-K&dateRange=custom&startdt={year_minus_2}-01-01&enddt={today}"`
   - EDGAR specific filing type: `curl -s -H "User-Agent: ResearchAgent (contact@example.com)" "https://efts.sec.gov/LATEST/search-index?q=%22{company_name}%22&forms={form_type}"`
   - ProPublica nonprofit lookup: `curl -s "https://projects.propublica.org/nonprofits/api/v2/search.json?q={org_name}"`

   Note: EDGAR EFTS returns filing metadata including filing URL. The discover skill should record the SEC filing URL as the source, status DISCOVERED. Actual content extraction happens in process-source.

4. **Credibility Tiers:**
   - Tier 1: Audited financial statements within SEC filings, 990 tax returns (legally mandated disclosures)
   - Tier 2: Non-audited SEC filings (8-K event reports, proxy statements), 990 schedules
   - Tier 3: SEC comment letters, no-action letters, regulatory guidance documents
   - Red flags: Amended filings (10-K/A) may indicate prior errors — check both original and amendment; late filings may indicate financial distress; 990s with large "other expenses" without itemization

5. **Source Status Taxonomy** — Reference web-search.md for canonical definitions. Note: SEC filings are always ACCESSIBLE (public by law). ProPublica 990 PDFs may require separate extraction.

6. **Degradation Behavior:**
   - Primary: Bash curl to EDGAR EFTS / ProPublica API
   - Fallback: `tavily_search` with `query: "{company_name} SEC filing {form_type}"`, `include_domains: ["sec.gov"]`, `search_depth: "advanced"`, `max_results: 5` — label as "via Tavily (EDGAR fallback)"
   - For nonprofits: `tavily_search` with `query: "{org_name} 990 filing nonprofit"`, `include_domains: ["propublica.org", "guidestar.org", "candid.org"]`, `max_results: 5`
   - Without Tavily: WebSearch with "site:sec.gov" or "site:propublica.org" — label accordingly
   - Unavailable criteria: HTTP 5xx, timeout > 15s, HTTP 429

7. **Rate Limits** — EDGAR: 10 requests/second max (SEC policy). ProPublica: undocumented, use 2 requests/second as safe default.

**Example Response section:**
Show trimmed example responses from both APIs:
- EDGAR EFTS: highlight `filings[].file_date`, `filings[].form_type`, `filings[].file_num`, `filings[].film_num`, and the filing URL construction pattern
- ProPublica: highlight `organizations[].name`, `organizations[].ein`, `organizations[].city`, `organizations[].state`, `organizations[].ntee_code`, and the 990 PDF URL construction pattern (`https://projects.propublica.org/nonprofits/organizations/{ein}`)

Create the domain-specific channel playbook. Per CONTEXT.md decision: this playbook uses a **template with type hooks** — structured placeholders per research type that Phase 2 type-channel maps will fill in with routing details.

**YAML frontmatter:**
- name: domain-specific
- description: Research-type-specific niche sources and specialized databases
- allowed-tools: [tavily_search, Bash, WebSearch]
- channel-type: domain-specific

**Use the identical section layout** (same 7 headers):

1. **Channel Overview** — Catches sources that don't fit the other 5 channels: patent databases, specialized industry databases, niche registries, standards bodies, educational repositories. This channel is type-dependent — different research types route to different specialized sources. The type-channel maps (Phase 2) specify which domain-specific sources apply to each research type.

2. **Tool Configuration** — Varies by source type:
   - Google Patents: URL construction + tavily_extract (no API, search via URL pattern)
   - Industry databases: tavily_search with domain scoping
   - Standards bodies: tavily_search or direct URL construction
   - Educational repositories: tavily_search with domain scoping

3. **Query Templates** — Organized as type hooks (per CONTEXT.md decision). Each hook is a block the discover skill selects based on research type:

   **Patent search (company, person, PRD validation):**
   - Google Patents URL: `https://patents.google.com/?q={search_terms}&assignee={company_name}&after=priority:{year_minus_5}0101`
   - The discover skill constructs this URL and uses `tavily_extract` to get results, OR presents the URL to the user

   **Industry databases (company, competitive analysis, market/industry):**
   - Crunchbase: `tavily_search` with `query: "{company_name} crunchbase"`, `include_domains: ["crunchbase.com"]`, `max_results: 3`
   - Industry reports: `tavily_search` with `query: "{industry} market report {year}"`, `include_domains: ["statista.com", "ibisworld.com", "grandviewresearch.com"]`, `max_results: 5`

   **Educational resources (curriculum research):**
   - Standards: `tavily_search` with `query: "{subject} standards framework"`, `include_domains: ["corestandards.org", "nextgenscience.org", "iste.org", "ed.gov"]`, `max_results: 5`
   - OER: `tavily_search` with `query: "{topic} open educational resource"`, `include_domains: ["oer.commons.org", "merlot.org", "openstax.org"]`, `max_results: 5`

   **Specialized registries (person research):**
   - Professional: `tavily_search` with `query: "{person_name} {field} profile"`, `include_domains: ["orcid.org", "researchgate.net", "scholar.google.com"]`, `max_results: 3`

   Note: These are starter hooks. Phase 2 type-channel maps determine which hooks each research type actually uses. The discover skill reads the type-channel map first, then selects the appropriate hooks from this playbook.

4. **Credibility Tiers:**
   - Tier 1: Patent office filings (USPTO, EPO), official standards documents, government educational databases
   - Tier 2: Established industry databases (Crunchbase verified data, Statista with cited sources), professional registries (ORCID)
   - Tier 3: User-submitted profiles, unverified database entries, industry reports without methodology disclosure
   - Red flags: Pay-to-play directories, vanity registries, industry reports that are actually vendor marketing, patent trolls (high volume, low specificity filings)

5. **Source Status Taxonomy** — Reference web-search.md for canonical definitions. Note: Patent filings are always ACCESSIBLE (public). Industry databases may be partially paywalled (DISCOVERED until verified).

6. **Degradation Behavior:**
   - This channel is inherently Tavily-based for most hooks, so degradation is simpler
   - If tavily_search unavailable: WebSearch with equivalent domain-scoped queries
   - If tavily_extract unavailable (for patent URLs): Present constructed URL to user for manual review
   - Label all fallback results with actual source method

7. **Rate Limits** — Varies by source. Tavily-based hooks: standard Tavily limits. Google Patents: no API rate limit (URL construction only). Industry databases: standard web rate limits apply.

**No example response section needed** for this channel (per CONTEXT.md discretion — Tavily response format is already known, and this channel is a routing template rather than a specific API).
